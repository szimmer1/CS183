URL's

Every request has a url with components

  • http = scheme
  • www.soe.ucsc.edu = host
  • /index.html = path
  • ?query=true = query parameters

Request methods

There are two main request methods

  • GET: Used to get a resource. A GET request should never result in a change on the server (may lead to race conditions)
    • Any parameters to a get request need to be part of the URL
  • POST: Can have an effect on the server: insert data, delete a record, etc.
    Requests can have:
    • Data payload
    • Dictionary of variables

A request is usually accompanied by request headers

Response status codes

Look it up on w3 specs
Of note, the server should respond with 303 status code and redirect path for a POST method; it is up to the browser to handle the redirect with a second, GET, request
