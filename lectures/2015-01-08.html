Python

pitfalls for new programmers

  • dynamically typed
  • Booleans are capitalized. Eg. c = False
  • Use indentation instead of braces. 
    if 'A' in dict:
    b=3
    return dict['A']+b
else:
    return False
  • note colons in control flow elements
  • 'A' in dict interpreted as conditional, i.e. True or False

functions

defined with def f(x,y): and return return (x+y)

str

declared as 'I am a string' or "I am a string"

  • to get utf-8 encoding (web2py) on ASCII string a, do a.encode('utf8')
  • formatted strings
    • syntax: "I am a formatted string with var %r" % (var)
    • %r is dynamically typed, interpreted at runtime

lists

defined as list = [1,3,4]

  • list have heterogeneous types
  • supported methods are append, insert, delete, slice, concat, iterate
  • use nested lists to implement a binary tree

tuple

defined as tuple = (1,2,4)

  • tuple is a list, but with immutable elements (no support for tuple[0] = 5
  • if a tuple element is an object, the object's properties are mutable

dictionaries

defined as dict = {'A':1,'B':2}

  • keys can be any type implementing __hash__, which is pretty much every data type
  • useful methods are has_key, keys, values, items
  • use dict of lists to implement a graph

classes

defined as class MyClass(object)

  • constructor
    • defined as def __init__(self,[properties])
    • assign properties as self.property = value

JSON

how to use the python json library

  • import json at file head
  • to encode in json, json.dumps(var)
