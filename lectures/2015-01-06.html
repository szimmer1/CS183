Chose web2py because:

  • Easy to learn
  • Concise
  • Uses python, which is good to access data structures, etc

How dynamic is a website built?

Scaling = multiple servers

  • Every request, once over, is over
  • Requests from the same browser, not guaranteed to go to same server
  • Cannot put data in the filesystem because it has limited sharing

The data lives in a database. Provides:

  • Persistence
  • Sharing across servers
  • Concurrency: servers can concurrently read/write data
  • Replication of the data in multiple databases, to guard from failure

Ingredients of a website/app

  • A framework for facilitating writing code
  • A database to store data
  • Code
  • Digital assets
  • Domain name, SSL cert
  • IP: copywrite, patents on tech, incorporating a company, other business-y things
  • Deployment environment
  • Monitoring of the website
  • Backups / data preservation
  • A marketing plan to make a web site known
  • Monetization / fundraising plan because in the end, keeping everything up at scale is not free

Editing applications

Registering apps, configuration, and editing MVC files all happens from the browser.

Routing

Url corresponds directly to MVC structure. For example:
https://app.domain.com/app/controller1/function1 corresponds in the MVC structure to

-> class app
        -> class controller1
                -> function1()
